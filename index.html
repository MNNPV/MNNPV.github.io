ABSTRACT

This research project aims to design and implement a modern, cloud-native Point of Sale (POS) system tailored for small-to-medium retail enterprises. The dissertation focuses on optimizing transaction latency, inventory synchronization, and user experience (UX) within a distributed architecture. The system utilizes a microservices approach to ensure scalability and fault tolerance.

PROJECT PROPOSAL

01. PROBLEM STATEMENT

Legacy POS systems suffer from monolithic architecture constraints, resulting in slow update cycles, poor hardware interoperability, and lack of real-time data analytics. Current solutions fail to address the specific needs of hybrid (online/offline) retail environments effectively.

02. OBJECTIVES

  • > Analyze legacy bottlenecks
  • > Develop API-first Architecture
  • > Implement Real-time Sync
  • > Conduct User Testing

03. SCOPE

  • > Inventory Management
  • > Transaction Processing
  • > Customer CRM Lite
  • > Reporting Module

04. METHODOLOGY

The project follows the Design Science Research Methodology (DSRM). Iterative development cycles will be used to create artifacts (code, diagrams, models), which will then be evaluated against defined performance metrics (latency, usability scores).
